## Sample User Stories

### US01 — Input student data
```
As a user,
I want to enter student details such as test scores, hours studied, etc.,
So that I can receive a performance prediction.
Acceptance Criteria:
- Input form accepts all required features.
- Validation for each field is present.
```

### US02 — View prediction results
```
As a user,
I want to see the predicted student performance,
So that I can take informed actions.
Acceptance Criteria:
- Results are displayed after form submission.
- Error message if prediction fails.
```

### US03 — Test model accuracy (QA)
```
As a QA Engineer,
I want to test the prediction output for sample datasets,
So that I can ensure the model is accurate.
Acceptance Criteria:
- MSE and accuracy can be measured and reported.
```
